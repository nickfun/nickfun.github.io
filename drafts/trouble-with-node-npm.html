Trouble with Node, NPM

Date Wed 05 March 2014 By Nick Fun

Problems

I've been having trouble getting Node and the related ecosystem to work, I thought I would document what I've been trying.

Goal

I have a simple Linode VPS running Ubuntu Server 12.04 LTS. I want to use grunt to concat and minify my javascript app, and also precompile my lodash templates. I do this at work on my Windows 7 workstation. I had hoped it would be easy to setup on my VPS so that I could develop side projects and play around with related tech.

Easy at first

Getting Node and npm installed on the machine was easy. I love Debian-based distros because you can just apt-get everything and it works great. So I installed Node and then npm using apt-get. Things seemed fine, but I could not install grunt-cli. The error messages that npm puts out are long and confusing as they include a stack trace.

The problem was that my version of Node was too old to run grunt. I had 0.6 installed and 0.8 is required for grunt.

Upgrade time

I went to the Node homepage and was happy to see that they provide linux binaries for download. /I don't have to compile from souce!/ I extracted the files and found the binary that I needed. I added a link to it in my path and tried it out. node --version reported that I am now running version 0.10. /Sweet/. But when I ran npm to install grunt, I was getting the same error.
